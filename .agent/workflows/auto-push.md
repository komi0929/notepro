# 自動GitHubプッシュ（恒久ルール）

## 原則

コード変更を行った後、**ユーザーの指示を待たずに** 自動でGitHubにプッシュする。

## タイミング

以下のいずれかが完了した時点で自動プッシュする：

- ファイルの作成・編集が一区切りついた時
- ビルドが成功した時
- バグ修正が完了した時
- 機能実装が完了した時

## 手順

// turbo-all

1. ステージング

```
git add -A
```

2. コミット（変更内容を簡潔に英語で記述）

```
git commit -m "feat/fix/refactor: 変更の要約"
```

3. プッシュ

```
git push origin main
```

## コミットメッセージ規則

- `feat:` — 新機能追加
- `fix:` — バグ修正
- `refactor:` — リファクタリング
- `style:` — UI/スタイル変更
- `chore:` — 設定・依存関係の変更
