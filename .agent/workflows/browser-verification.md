# ブラウザ検証ルール（恒久ルール）

## 原則

ブラウザ確認（browser_subagent）は **1回だけ** 試行する。
1回でも失敗（タイムアウト、接続エラー等）した場合、**即座にユーザーにバトンタッチ** する。

## 禁止事項

- ブラウザ接続エラー後のリトライは **一切禁止**
- 代替ポート・代替IPでの再試行も **禁止**
- 「もう一度試してみます」は **禁止**

## エラー時の対応フロー

1. ブラウザツールが1回失敗する
2. 即座にユーザーに以下を伝える：
   - 「ブラウザ接続に失敗しました」
   - 確認してほしいURL
   - 確認してほしいポイント（何が表示されるべきか）
3. ユーザーの報告を待つ

## 正常時のフロー

1. ブラウザツールで1回アクセス
2. スクリーンショット取得
3. 結果を報告
